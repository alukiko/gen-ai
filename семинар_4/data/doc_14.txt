Добавлены два набора исполнителей для одного LangGraph-пайплайна (economic_7): встроенные агенты (builtin) и роли из MAR (mar_roles). Переключение — через executor_set в конфиге эксперимента; оркестрация (планировщик → oracle → семь ролей) остаётся той же.
Отдельно добавлен режим mar_catalog — это уже другой граф, а не третий executor_set. Планировщик (planner_mar) выбирает роли из всего каталога MAR/Roles/** (Code, Math, Commonsense, FinalNode и др.), а не из фиксированных семи. Oracle (oracle_mar) распределяет бюджет с учётом домена роли. Все шаги выполняет один универсальный узел mar_executor: подставляет нужный JSON-профиль роли, промпты MAR и LLM; при необходимости подключаются поиск (Tavily) или Python. Для файлов/медиа может отработать document_media_agent. Финальный ответ собирает узел finish (предпочитая роли из FinalNode). Параметр executor_set здесь не влияет на граф.
В minimal_executor_backends_10.ipynb — примеры запуска на GAIA: сравнение builtin и mar_roles на economic_7 и отдельный пример с pipeline="mar_catalog".
Имена папок с результатами теперь включают политику, бюджет, pipeline и (для economic_7) тип исполнителей; для mar_catalog сегмент exec- не добавляется — проще отличать прогоны друг от друга.
Результаты залиты в репозиторий в отдельную ветку
https://github.com/tail-call/economic-agents/tree/feature/mas-router-executors
